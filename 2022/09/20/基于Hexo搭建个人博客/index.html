基于Hexo+GitHub Pages搭建个人博客

发表于 2022-09-20 分类于 Tools ， Hexo Valine：
本文字数： 6.9k 阅读时长 ≈ 6 分钟

本博客详细地介绍了如何使用 Hexo 搭建属于自己的个人博客，并部署到 **Github Pages **上。如有需要，可以访问本人 GitHub 地址

前言

Hexo 是一个快速、简洁且高效的静态博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，即将 .md 的文件转换为 html 静态网页文件。由于目前 Hexo 在网上教程比较多，遇到问题也比较好找到解决方案，所以对于想要快速搭建个人博客的新手来说 Hexo 是博客搭建的首选框架。有关 Hexo 的更多介绍，可以浏览 Hexo官网 获取相关信息。

本教程详细地记录了博主搭建博客的全过程，遇到的问题以及解决办法，此外还列举了很多新手容易犯错的地方…… 如果大家能跟着本教程一步步执行下去，那么大家就可以成功的从 0 到 1 搭建一个自己的博客。如果你还是一个喜欢折腾的人，在此基础上你还可以优化你的博客，这样你就可以拥有一个独特，拥有个性化，世界上独一无二的专属博客啦 ！

说明

因为本教程篇幅有限，身处网络如此发达的今天，对于本教程涉及的一些工具或专业术语的详细介绍大家在网上都能找到很好的解答，因此我就不一一详细介绍了。我想大家更关注的应该是如何实现个人博客搭建这个目标，但我会提供相应的官网地址，有兴趣的读者可以自行前往了解。

部署到 GitHub Pages

博客的目的就是记录和共享，当然是希望能有人能看见自己写的内容了；而本地的博客就只能本地启动，总不能自己搭个本地博客自己看吧，那这样也就不需要搭建博客了，Windows的资源管理器就能做到这一点。因此，我们希望搭建的博客能部署到别的平台上去，这里有很多选择，比如 Github，Gitee，CSDN 等…… 而博主选择的就是将博客部署到 GitHub 上去，虽然 GitHub 可能有国内访问慢或直接访问不了等问题，但是作为全球最大的代码托管平台，其重要地位不言而喻。

创建仓库

首先需要在 GitHub 上有一个自己的账号，没有账号的需要自己先注册一个。然后在自己的账号下新建一个和你用户名相同的仓库。比如我的账号名为 Teksab，那么仓库名就需要设为 Teksab.github.io，只有这样博客部署到 GitHub Pages 上才会被识别。

执行一次 hexo clean 命令，会直接清空缓存文件 (db.json) 和删除 public 文件夹。执行一次 hexo g 命令，又会重新生成 public 文件夹，而我们网站的每一个页面都对应着一个 index.html 的静态文件，存放在 public 文件夹的子文件夹下，这些静态文件都是经过 Hexo 将 .md 的文件渲染而来。执行一次 hexo d 就是完成一次部署，其实就是将站点文件夹下的 public 夹下的内容转移到到仓库中，这样我们就可以访问远程仓库中的静态页面了，这也就构成了我们的博客了。

如果我们修改了文章内容，添加了新文章或修改了主题配置文件等信息后，需要在站点目录下依次执行 hexo clean，hexo g，hexo d 这三条命令，重新部署我们的博客到远程仓库。

GitHub 绑定个人域名

其实完成以上的内容，基本已经完成了博客的搭建流程了。但如果你觉得每次使用网址 https://yourname.github.io 来访问博客不够上档次，这里还可以绑定自己的域名。前提是你得有一个域名，可以去买域名，或者去哪嫖一个域名，再或者这一步就可以直接跳过（域名也就是使访问博客的时候更高大上一点，不要也罢）。因此，本小节的内容针对的是已经有域名的小伙伴。

首先准备一个域名：什么 xxx.com，xxx.cn，xxx.top 都行，博主的域名就是阿里云的 .cn 域名，以下示例展示的就是按照绑定阿里云上的域名展开的，购买其他平台的域名操作按理来说大同小异。

在域名解析设置中，我们需要添加两条解析记录，如下图所示：

  1. 添加的一条记录：主机记录为 www，记录类型选择 CNAME，解析路线选择默认，记录就是 yourname.github.io 。

打开 CMD（快捷键：“Win键+R”->cmd，回车），ping 一下GitHub Pages上博客地址，也就是 yourname.github.io。其中，来自后面的那串 ip 就是你博客的 ip 地址。

在 GitHub 的仓库中，点击 Settings -> Pages -> Custom domain ,在 Custom domain 中输入你的域名，这个域名就是在平台上购买的域名，前面不要加像 blog. 等前缀。点击 Save，完成域名绑定。

在站点的 source 文件夹中添加 CNAME 文件的原因：每一次执行 hexo d 命令重新将本地博客部署到远程时，会删除仓库的所有内容，包括 CNAME 文件，而 CNAME 文件就是仓库绑定域名的纪录。删除 CNAME 文件的同时就相当于仓库和域名解绑，那么下一次重新部署时就需要重新绑定。而向 source 中添加 CNAME 文件后，每一次部署操作就会重新上传 CNAME 文件，这样就不需要每次重新部署再绑定域名了。

写文章

正如前面介绍所知，Hexo 写文章采用的是 Markdown 编写。Markdown 是一种轻量级的标记语言，可用于在纯文本文档中添加格式化元素。Markdown 由 John Gruber 于 2004 年创建，如今已成为世界上最受欢迎的标记语言之一。

  1. 专注于文字内容；
  2. 纯文本，易读易写，可以方便地纳入版本控制；
  3. 语法简单，没有什么学习成本，能轻松在码字的同时做出美观大方的排版。

关于 Markdown 的更多介绍可以访问 Markdown 官方教程 或者我的另一篇博客：Markdown 写文章教程

而对于编写 Markdown 的编辑器，使用的最多也是本人在使用的就是 Typora。如有需要，大家也可以去 Typora 中文官网 下载。但是由于去年 Typora 开始收费，因此也有一部分用户选择转向其他免费的 Markdown 编辑器。其实到底选择哪个编辑器不重要，重要的是写出一篇好文章是嘛！
